# Store image attachments for history and agent access

Carrent stores image attachments as stable files in the app data attachment store instead of embedding image data in `workspace.json` or writing hidden attachment directories into user projects. This keeps conversation history viewable without bloating workspace state, and matches the agent GUI boundary: Carrent owns chat attachments in a separate scope, preserving the user's image format, project contents, and visible message history as much as possible. Runtimes receive native image parts when supported, with Carrent-managed local paths used only as needed for runtime access. Attachment files are retained while their messages remain in history; automatic expiry is deferred until Carrent has message deletion or archive cleanup semantics.
