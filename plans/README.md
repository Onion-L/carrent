## Findings considered and rejected

- `RuntimeProcessManager` can let an old child exit callback delete a newer
  child entry. The lifecycle IPC remains exposed, but the current renderer does
  not call process start/stop/restart; the visible runtime controls only update
  enabled settings. Re-audit when lifecycle controls are reconnected.
- Local MCP has a start-stop-start generation race, but the only current UI
  serializes toggles with a loading guard and application initialization
  completes before the window is created. Re-audit if multiple MCP controls or
  background restart behavior are added.
- `shell.openExternal` has no main-process scheme allowlist. Current untrusted
  links pass through react-markdown's safe URL transform, which rejects custom
  protocols. Add a main-process allowlist if another untrusted window-open
  source is introduced.
- `bun audit` reported 25 advisories, including 10 high severity advisories.
  The quick audit did not verify runtime reachability: the landing app is a
  static empty page and the remaining high advisories are on development or
  packaging paths. Do not file blanket upgrades without a reachability check.
