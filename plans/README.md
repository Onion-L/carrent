- Within Plan 001, implement fresh-session transcript replay before routing
  user-message replacement runs through a fresh Kimi session. The replacement
  path depends on the replay behavior to reconstruct the intended history.
- Plan 003 intentionally owns all persistent deletion work. Do not split its
  renderer pruning, provider-session invalidation, and attachment deletion into
  independently shipped changes; partial deletion would preserve the original
  bug under a different shape.

  cache. It changes only the active Runtime Settings presentation and must not
  reconnect the unused standalone `RuntimesPage`.
- Plan 005 treats a successful ACP session without model options as Ready and
  leaves model selection to Kimi's runtime-owned default.
- Plan 006 stores a bounded workspace-versus-HEAD snapshot after a project Run.
  It deliberately does not attribute every change to the Coding Agent and must
  remain read-only; exact before/after Run attribution is deferred.
- Plan 007 depends on Plan 006's stored snapshot and viewer. It adds a transient,
  user-confirmed follow-up draft only; it must not change snapshot persistence,
  query Git, auto-send a message, or add workspace mutation controls.
- Plan 008 uses Kimi's native ACP `plan` mode and runtime-owned Plan Review. It
  must not inject planning instructions into prompts or broaden external file

## Findings considered and rejected

- `RuntimeProcessManager` can let an old child exit callback delete a newer
  child entry. The lifecycle IPC remains exposed, but the current renderer does
  not call process start/stop/restart; the visible runtime controls only update
  enabled settings. Re-audit when lifecycle controls are reconnected.
- Local MCP has a start-stop-start generation race, but the only current UI
  serializes toggles with a loading guard and application initialization
  completes before the window is created. Re-audit if multiple MCP controls or
  background restart behavior are added.
- `shell.openExternal` has no main-process scheme allowlist. Current untrusted
  links pass through react-markdown's safe URL transform, which rejects custom
  protocols. Add a main-process allowlist if another untrusted window-open
  source is introduced.
- `bun audit` reported 25 advisories, including 10 high severity advisories.
  The quick audit did not verify runtime reachability: the landing app is a
  static empty page and the remaining high advisories are on development or
  packaging paths. Do not file blanket upgrades without a reachability check.
