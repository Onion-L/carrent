# Support file attachments as read-only snapshots

Carrent now supports one or more mixed Image and UTF-8 File Attachments per message. Every attached file is copied into Carrent's app-data attachment store as a stable snapshot, so history stays reproducible and raw app-data paths stay out of visible messages and `workspace.json`. Images keep their native ACP image blocks, thumbnails, and lightbox behavior. File Attachments are sent to Kimi as ACP `resource_link` blocks and are readable by the Runtime only through the exact canonical paths present on the current `ChatTurnRequest` — read-only, with no writes, no sibling files, and no directory-wide grants. Folders remain additional directories, not attachments. Binary non-image formats (PDF, Office documents, audio, video, archives) stay unsupported until the primary Runtime exposes a reliable binary/document input path. This decision extends ADR 0003's Carrent-owned snapshot model to text files and supersedes only the single-file cardinality wording in ADR 0007.
